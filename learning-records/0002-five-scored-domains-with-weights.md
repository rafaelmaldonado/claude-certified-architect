## Context
Lesson 1 and the initial mission were built on the public Skilljar page, which lists
**4 core technologies** (Claude API, MCP, Claude Agent SDK, Claude Code). The user
corrected this: the Exam Guide PDF defines **5 scored competency domains** with weights.

## The real blueprint (from the user's Exam Guide PDF)
| # | Domain | Weight |
|---|--------|-------|
| 1 | Agentic Architecture & Orchestration | 27% |
| 2 | Claude Code Configuration & Workflows | 20% |
| 3 | Prompt Engineering & Structured Output | 20% |
| 4 | Tool Design & MCP Integration | 18% |
| 5 | Context Management & Reliability | 15% |

## Key insight
The 4 "technologies" are the **material**; these 5 are how mastery is **scored**. The exam
is competency-based, and **Agentic Architecture & Orchestration (27%)** dominates — which
confirms the "tradeoff decisions" positioning. Teaching effort should be weighted accordingly,
not split evenly.

## Implications
- Re-anchor all navigation/lessons on the 5 weighted domains.
- Prioritize time by weight; Agentic Architecture gets the most lessons.
- Lesson 1's "which surface?" ladder is actually core to Domain 1 (27%) — good, keep and deepen it.
- Still need Q count / time / pass score from the PDF to calibrate quiz format.
