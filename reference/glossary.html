Course Glossary

The shared vocabulary of the four exam domains. Every lesson uses these terms exactly as defined here. This is the canonical spelling of each concept — revisit it often.

Adhere to this Once a term is defined here, all lessons use it consistently. If a lesson introduces a new term, it gets added here.

The four exam domains

DomainOne-line meaning
Claude APIThe /v1/messages HTTP endpoint (and its SDKs) for sending prompts and getting responses. Everything else builds on it.
Model Context Protocol (MCP)An open standard for connecting Claude to external tools/data via reusable "servers" — like a USB port for AI capabilities.
Claude Agent SDKClaude Code packaged as a library: a batteries-included coding/filesystem agent (built-in Read/Write/Bash/etc.) you run on your own infra.
Claude CodeAnthropic's agentic CLI/IDE tool for software work — the end-user product built on the same primitives.

Core API concepts

TermDefinition
Messages APIThe single endpoint POST /v1/messages. Tools, structured output, thinking, and caching are all features of this one endpoint, not separate APIs.
StatelessThe API keeps no memory between calls. You resend the full conversation history every request.
System promptTop-level instructions that set the model's role/behavior, separate from the user turns.
Tool useThe model requests a function call (a tool_use block); your code runs it and returns a tool_result. Basis of all agents.
Tool RunnerAn SDK helper that automates the request → run tool → feed result → repeat loop for tools you define. Harness only; you host it.
Adaptive thinkingthinking: {type:"adaptive"} — the model decides when/how much to reason. Replaces the old fixed budget_tokens.
Effortoutput_config.effort (low→max): dials thinking depth and total token spend. The main cost/quality lever.
Prompt cachingReuse a stable prompt prefix across requests (~90% cheaper on the cached part). A prefix match — any byte change invalidates everything after it.
Structured outputsConstrain the response to a JSON schema (output_config.format) so it's guaranteed parseable.
StreamingReceive the response incrementally as Server-Sent Events. Required for large max_tokens to avoid timeouts.
stop_reasonWhy the model stopped: end_turn, max_tokens, tool_use, pause_turn, refusal.

Agent & orchestration concepts

TermDefinition
WorkflowMulti-step pipeline where your code controls the flow. Predictable; you decide each step.
AgentOpen-ended, model-driven tool use — the model decides the steps. Use only when the task truly needs it.
Managed Agents (CMA)Anthropic hosts both the agent loop and a per-session sandbox. Persisted, versioned agent configs + sessions.
Harness vs deploymentHarness = the agent loop + context management. Deployment = the infra it runs on. Managed Agents supplies both; Tool Runner & Agent SDK supply harness only.
MCP serverA process exposing tools/resources/prompts over MCP. Reusable across any MCP client (Claude Code, the API's MCP connector, etc.).
