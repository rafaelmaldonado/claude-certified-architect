One call, one loop — and whose computer runs it

Your win today: say out loud, for each of the four approaches, who writes the loop and whose computer it runs on — and see the Managed Agents version of the same add-tool example you already met in Lesson 2.

Your own question, answered “Is it a code loop that runs in different environments?” Yes. That's the whole idea. The agent loop from Lesson 2 is one piece of code. The four approaches only differ by two things: who writes that loop and whose computer it runs on.

1 · “Single API call” is the thing before a loop

A single API call is one round-trip to /v1/messages: you send messages, you get one response. Nothing loops. If the model replies with stop_reason: "tool_use", nothing automatically runs the tool — the response just sits there.

An agent is what you get when you wrap a loop around that single call: run the tool, feed the result back, call again, repeat until end_turn. So:

single API call   = one turn.           (no tools run for you)
agent             = a loop of API calls. (tools run, results fed back)

Everything in Lesson 2 was about who builds that loop. This is the same skill, said slowly.

2 · The two questions, drawn as a map

Only two axes matter. Read each row as “who writes the loop code / whose computer runs it”:

Approach Who writes the loop Whose computer Built-in tools?
Single API call — (no loop) yours none — one turn only
Manual loop you, by hand yours only tools you define
Tool Runner the SDK (a helper) yours only tools you define
Claude Agent SDK the SDK (full harness) yours Read/Write/Bash/Grep + MCP
Managed Agents Anthropic Anthropic's hosted sandbox (bash/files/code) + MCP + yours
The one line that separates them Manual loop, Tool Runner, and Agent SDK all run on your computer — they only differ in how much loop code you write yourself (all of it → some of it → almost none). Managed Agents is the only one that also runs on Anthropic's computer.

3 · The same “What is 2 + 3?” task — now as a Managed Agent

In Lesson 2 you saw this task as a manual loop (you write all five beats) and as the Tool Runner (the SDK writes them, on your machine). Here's the third view: Managed Agents, where Anthropic writes the loop and runs it on a session in their sandbox.

# 1. Create a persisted agent config ONCE (server-side; versioned, reusable).
agent = client.beta.agents.create(
    model="claude-opus-4-8",
    name="calculator",
    instructions="You do arithmetic using the add tool.",
    tools=[{
        "name": "add",
        "description": "Add two integers.",
        "input_schema": {"type": "object",
            "properties": {"a": {"type": "integer"}, "b": {"type": "integer"}},
            "required": ["a", "b"]},
    }],
)

# 2. Open a session and send the task. Anthropic runs the loop for you.
session = client.beta.agents.sessions.create(agent_id=agent.id)
run = client.beta.agents.sessions.runs.create(
    session_id=session.id,
    input="What is 2 + 3?")

# 3. The run PAUSES when it needs YOUR tool (add isn't in the sandbox).
#    You supply the result; Anthropic resumes the loop on its side.
while run.status == "requires_action":
    outputs = []
    for call in run.required_action.tool_calls:      # same tool_use idea…
        result = run_tool(call.name, call.input)
        outputs.append({"tool_call_id": call.id,     # …same "echo the id" rule
                        "output": result})
    run = client.beta.agents.sessions.runs.submit_tool_outputs(
        run_id=run.id, tool_outputs=outputs)

print(run.output)   # "2 + 3 = 5"
Look at what changed — and what didn't Didn't change: tools are still declared, and every result still echoes the id of the call it answers (tool_call_id here = the tool_use_id from Lesson 2). Changed: there is no while that re-calls the model — you don't resend the conversation, you don't check end_turn. Anthropic's server owns the loop and the state. You only step in to run your own tools; its built-in sandbox tools (bash, files) run on their side without ever pausing.
Why the pause exists Your add function lives on your computer, so the server can't run it — it pauses and asks you. If the agent had used a sandbox tool (run some Python, read a mounted file), no pause: that runs on Anthropic's side. The pause is exactly the seam between “their computer” and “yours.”

Note: exact method names for the Managed Agents beta move fast — treat this as the shape, not the syntax, and check the docs. The concept (persisted agent → session → run that pauses for your tools) is what the exam tests.
