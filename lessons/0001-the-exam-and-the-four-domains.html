Lesson 1 · Orientation + first tradeoff skill

The exam, the four domains, and “which surface?”

Your win today: name the five scored exam domains (and which weighs most), and correctly pick which Claude “surface” a given job needs. That second skill is the exam’s whole personality — it grades tradeoff decisions, not trivia.

What this certification actually tests

The Claude Certified Architect – Foundations exam ($125, Anthropic partner program) validates one thing above all:

You can “make informed decisions about tradeoffs when implementing real-world solutions with Claude.”

Read that twice. It is not “can you recite the API.” It is “given a real problem, do you pick the right tool and the right settings, and can you say why.” Every lesson in this course is built around that verb: decide.

Material vs. scoring Under these five domains sit four technologies — the Claude API, MCP, the Claude Agent SDK, and Claude Code. The technologies are what you learn; the five domains are how you’re graded. They stack: API → (tools) → MCP → Agent SDK → Claude Code. Full definitions in the glossary.
The one mental model Nearly every architecture question reduces to: how much control does the model have, and who runs the loop? A single API call gives the model zero autonomy and you total control. A full agent gives the model the wheel. The art is choosing the least-powerful option that still does the job.

The decision ladder: which surface for the job?

This is the single most exam-relevant skill. Start at the top and stop at the first row that fits — don’t reach for an agent when a single call will do.

If the job is…UseWhy
Classify / summarize / extract / answer — one shotSingle API callOne request in, one response out. No loop needed.
Multi-step, but you know the steps in advanceWorkflow (API + tool use, your loop)Predictable. Your code controls flow; cheaper and easier to debug.
Open-ended; the model must decide the steps, your own toolsAgent (API + Tool Runner)Model-driven. You host it; SDK runs the tool loop.
Agent that needs a hosted sandbox / persisted, versioned configManaged AgentsAnthropic runs the loop and the container. Least code you own.
A coding/filesystem agent on your infra, batteries includedClaude Agent SDKShips built-in Read/Write/Bash/etc. + the full harness.
You (a developer) working in a terminal / IDEClaude CodeThe end-user product; not something you embed.
Classic exam trap “Tool Runner” and “Claude Agent SDK” sound alike but differ. Tool Runner is a thin helper in the regular API SDK that loops over tools you define — no built-in tools. The Agent SDK is Claude Code as a library with built-in file/bash tools. Both are harness-only (you deploy them). Only Managed Agents also hosts the deployment.

The four “should I even build an agent?” checks

Before choosing any agent row, the exam expects you to sanity-check all four:

  • Complexity — is the task genuinely multi-step and hard to script in advance?
  • Value — does the outcome justify higher cost and latency?
  • Viability — is Claude actually good at this task type?
  • Cost of error — can mistakes be caught and recovered (tests, review, rollback)?

A “no” on any of these means step back down the ladder to a workflow or a single call.
